# 商业上线就绪度

本文以代码、数据库迁移、自动化测试和浏览器验收为证据，不把仅存在于页面或 Schema 中的设计计为已完成能力。

## 当前结论

核心单团队研发闭环已经可运行，但整个产品尚未达到“无需附加工作即可公开商业上线”的标准。建议当前定位为内部试运行或受控 Beta，完成下方 P0 项后再进入正式生产发布。

| --- | --- | --- |
| 需求 | CRUD、服务端分页筛选、状态机、乐观锁、历史、审计、附件、领域事件和通知 | 迭代管理 UI/API、批量导入的幂等/事务语义、多人实时更新、关键前端组件测试 |

## P0 生产门禁

1. PostgreSQL 自动备份、时间点恢复和实际恢复演练。
2. 已提供受独立令牌保护的 Outbox、死信、数据库连接和容量指标；上线环境仍须接入 Prometheus/告警平台，并补齐 API 延迟、Worker 存活、磁盘和容器指标。
3. 三个机器人渠道使用真实企业租户完成签名、卡片渲染、限流和错误码契约测试。
4. 登录、RBAC 越权、需求流转、缺陷闭环、用例执行、附件和通知的独立环境 E2E。
5. 批量导入幂等键、逐行结果和安全重试，避免网络中断后重复创建。
6. 完成 SSO/MFA 或明确首个商业版本不承诺这些能力并从产品说明中移除。
7. 完成容量测试：100–200 并发用户、Outbox 峰值、脑图大数据量和对象存储吞吐。

## 数据清洁规则

- `prisma/seed.ts` 只维护权限目录，以及不可删除的组织管理员和项目空间管理员；业务角色由各组织自行创建。
- `prisma/bootstrap.ts` 只在显式执行时创建首位系统管理员，以及可编辑的默认组织和默认项目空间；不创建任何需求、缺陷、用例或示例成员。
- 生产部署不得执行 E2E 脚本，不得携带示例组织、空间、需求、缺陷或用例。
- 浏览器存储只允许保存脑图折叠等界面偏好，不得成为业务事实源。
